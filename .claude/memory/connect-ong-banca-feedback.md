Evaluation board (banca) feedback on the first delivery:

**Praised:** well-defined architecture; frontend↔backend↔DB integration; relevant social proposal; consistent functioning.

**Main criticism (the gap to close):** the system has **little interaction**, lacks **continuous engagement**, has **no communication between doador and ONG**, and needs a **more dynamic experience**.

**Explicit suggestion from the banca:** implement a **"match" system** between donations and ONG needs, with an **automatic chat opened after acceptance**.

**Why this matters:** this was the professors' advice to make the project feel alive — NOT a guaranteed way to win (the user corrected my earlier framing; see [[connect-ong-milestones]]). Still, it's the clearest direction to address the "no interaction / no communication" gap. The hero feature for the showcase should be the **Match → Chat** flow (ONG publishes a need → doador shows interest → ONG accepts → a connection + real-time chat is created). Everything else (dashboard of impact, feed of needs, verification, payments/PIX, JWT, etc.) is in the user's roadmap but must be prioritized ruthlessly behind this. See [[connect-ong-vision]], [[connect-ong-milestones]], [[connect-ong-tech-guidelines]].
